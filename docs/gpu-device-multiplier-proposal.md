# GPUFabric 设备倍率方案

## 📋 概述

基于 io.net 的分层奖励结构，为 GPUFabric 设计一套合理的设备倍率方案，旨在优化网络算力分布，激励高质量设备参与。

## 🎯 设计原则

1. **性能导向**：基于 TFLOPS 性能指标分级
2. **分层管理**：企业池、社区池、不支持三层结构
3. **灵活调整**：可根据市场需求动态调整倍率
4. **向后兼容**：新设备采用默认倍率，逐步优化

## 🏷️ 设备分类体系

### 🟢 企业池（高端设备）
- **目标**：吸引数据中心级 GPU
- **倍率范围**：1.0 - 6.0
- **典型设备**：H100、A800、H200、B100、RTX 5090

### 🟡 社区池（中端设备）
- **目标**：维持主流 GPU 参与度
- **倍率范围**：0.3 - 0.8
- **典型设备**：RTX 5080、RTX 5070、RTX 4080、RTX 4070

### 🔴 不支持（低端设备）
- **目标**：逐步淘汰老旧设备
- **倍率范围**：0.1 - 0.2
- **典型设备**：RTX 5060、AMD 移动端 GPU

## 🛠️ 实施方案

### 1. 数据库更新脚本

```sql
-- 批量更新设备倍率（参考 io.net 企业池标准）
UPDATE device_types SET points_multiplier = 
  CASE device_id
    -- 顶级企业设备（参考 io.net 企业池最高倍率）
    WHEN 30 THEN 6.0    -- H200 (最新旗舰)
    WHEN 31 THEN 6.0    -- B100 (Blackwell 架构)
    WHEN 8965 THEN 5.0  -- H100 80GB HBM3
    WHEN 8961 THEN 5.0  -- H100 SXM
    
    -- 高端企业设备
    WHEN 8960 THEN 4.0  -- H100 PCIe
    WHEN 8954 THEN 3.5  -- H100 NVL
    WHEN 8435 THEN 2.5  -- A800 SXM4 80GB
    WHEN 8437 THEN 2.5  -- A800 80GB PCIe
    WHEN 8438 THEN 1.5  -- A800 40GB PCIe
    
    -- 消费级旗舰
    WHEN 11141 THEN 1.5 -- RTX 5090
    WHEN 11143 THEN 1.2 -- RTX 5090 D
    WHEN 11289 THEN 0.7 -- RTX 5090 Max-Q / Mobile
    
    -- 社区池设备
    WHEN 11266 THEN 0.8 -- RTX 5080
    WHEN 11288 THEN 0.6 -- RTX 5080 Max-Q / Mobile
    WHEN 12036 THEN 0.5 -- RTX 5070
    WHEN 11269 THEN 0.5 -- RTX 5070 Ti
    WHEN 12056 THEN 0.4 -- RTX 5070 Ti Mobile
    WHEN 10245 THEN 0.4 -- RTX 4060 Ti 16GB
    WHEN 10248 THEN 0.3 -- RTX 4060
    WHEN 10400 THEN 0.3 -- RTX 4060 Max-Q / Mobile
    WHEN 10144 THEN 0.6 -- RTX 4080 Max-Q / Mobile
    WHEN 10272 THEN 0.4 -- RTX 4070 Max-Q / Mobile
    
    -- Apple M 系列（社区池）
    WHEN 9001 THEN 0.6 -- Apple M4 (GPU)
    WHEN 9002 THEN 0.5 -- Apple M3 (GPU)
    WHEN 9003 THEN 0.4 -- Apple M2 (GPU)
    WHEN 9004 THEN 0.3 -- Apple M1 (GPU)
    
    -- 不支持设备
    WHEN 11524 THEN 0.2 -- RTX 5060 Ti
    WHEN 11525 THEN 0.2 -- RTX 5060
    WHEN 11545 THEN 0.15 -- RTX 5060 Max-Q / Mobile
    WHEN 5510 THEN 0.1  -- AMD Ryzen AI Max+ 395
    
    -- 默认新设备倍率
    ELSE 0.05
  END,
  updated_at = NOW()
WHERE device_id IN (
  -- 企业池设备 ID
  30, 31, 8965, 8961, 8960, 8954, 8435, 8437, 8438, 11141, 11143, 11289,
  -- 社区池设备 ID
  11266, 11288, 12036, 11269, 12056, 10245, 10248, 10400, 10144, 10272,
  -- Apple M 系列 ID
  9001, 9002, 9003, 9004,
  -- 不支持设备 ID
  11524, 11525, 11545, 5510
);
```

### 2. 质押要求（可选扩展）

参考 io.net 质押模型：

```
基础质押：100 GPUFabric Tokens
质押计算 = 基础质押 × 倍率 × GPU数量（参考 io.net 模型）

示例：
- 8 × H100 (倍率 5.0)：100 × 5.0 × 8 = 4,000 Tokens
- 4 × A800 (倍率 2.5)：100 × 2.5 × 4 = 1,000 Tokens
- 2 × RTX 5090 (倍率 1.5)：100 × 1.5 × 2 = 300 Tokens
- 4 × RTX 5080 (倍率 0.8)：100 × 0.8 × 4 = 320 Tokens
- 1 × Apple M4 (倍率 0.6)：100 × 0.6 × 1 = 60 Tokens
- 1 × Apple M3 (倍率 0.5)：100 × 0.5 × 1 = 50 Tokens
- 1 × Apple M2 (倍率 0.4)：100 × 0.4 × 1 = 40 Tokens
- 1 × Apple M1 (倍率 0.3)：100 × 0.3 × 1 = 30 Tokens
- 2 × RTX 5060 (倍率 0.2)：100 × 0.2 × 2 = 40 Tokens
```
