## Release Artifact Integrity

Run the release evidence helper before publishing. Pull-request CI may omit artifacts and only generate a status note:

```bash
scripts/security_release_evidence.sh security-release-evidence <artifact-dir>
```

Release jobs must run strict mode so missing artifacts or missing signing proof fail the gate:

```bash
GPUF_REQUIRE_ARTIFACTS=1 \
GPUF_REQUIRE_SIGNING=1 \
GPUF_SIGNING_TOOL=cosign \
COSIGN_KEY=<cosign-key-ref> \
scripts/security_release_evidence.sh security-release-evidence <artifact-dir>
```

Use `GPUF_SIGNING_TOOL=minisign` with `MINISIGN_KEY`, or `GPUF_SIGNING_TOOL=gpg` with the release GPG identity, if those are the selected release signing mechanisms.

Attach `SHA256SUMS` for every SDK archive, `.so`, `.a`, `.aar`, `.xcframework`, and installer payload. Attach `release-policy.txt`, `signing-tool.txt` or `signing-status.txt`, and any signature file generated by the helper.

## Mobile SDK Release Evidence

Default CI records mobile SDK release-gate status without requiring Android/iOS devices:

```bash
scripts/mobile_sdk_release_gate.sh security-release-evidence/mobile-sdk
```

Formal mobile SDK release jobs must attach platform evidence and fail if it is incomplete:

```bash
GPUF_REQUIRE_MOBILE_EVIDENCE=1 \
scripts/mobile_sdk_release_gate.sh security-release-evidence/mobile-sdk <mobile-evidence-dir>
```

The evidence directory must contain non-empty files named `android-target-compile.txt`, `ios-target-compile.txt`, `android-keystore-review.txt`, `ios-keychain-review.txt`, `mobile-control-tls-pinning.txt`, `android-instrumentation.txt`, `ios-simulator-device.txt`, `asan-tsan.txt`, and `permissions-logging-audit.txt`. These files should link to device/simulator logs, CI job URLs, or signed-off review notes.
