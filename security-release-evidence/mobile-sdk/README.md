# GPUFabric Mobile SDK Release Gate Evidence

Generated by `scripts/mobile_sdk_release_gate.sh`.

Default CI mode records mobile release gate status. Formal mobile SDK release jobs must set `GPUF_REQUIRE_MOBILE_EVIDENCE=1` and pass an evidence directory containing:

- `android-target-compile.txt`
- `ios-target-compile.txt`
- `android-keystore-review.txt`
- `ios-keychain-review.txt`
- `mobile-control-tls-pinning.txt`
- `android-instrumentation.txt`
- `ios-simulator-device.txt`
- `asan-tsan.txt`
- `permissions-logging-audit.txt`

Files in this bundle:

- `mobile-release-context.txt`: git and gate context.
- `mobile-source-sensitive-grep.txt`: forbidden mobile source-pattern grep output; any match fails the gate.
- `mobile-toolchain-status.txt`: local availability of Android/iOS diagnostic tools.
- `mobile-sdk-symbols.txt`: SDK compatibility and control TLS reference search.
- `mobile-evidence-status.txt`: required release evidence presence check.
- `mobile-release-gate-status.txt`: present only when a required gate failed.
