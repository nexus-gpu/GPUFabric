Findings:
  The native iOS C SDK exposes runtime APIs but does not implement persistent credential storage.
  No Keychain usage was found in the SDK or simulator sample.
  No UserDefaults / NSUserDefaults storage was found in the iOS simulator sample.
  No plaintext token/credential file write was found in the iOS simulator sample.

Security conclusion:
  The SDK does not persist long-lived token or credential material on iOS.
  Production iOS app wrappers must store long-lived credentials in Keychain, not UserDefaults, logs, plist files, or arbitrary Documents files.
  SDK callers should pass only runtime-scoped values into start_remote_worker/start_remote_worker_with_tls.

Logging caveat:
  The simulator sample logs callback text, model path, and test client identifier with public privacy for diagnostics.
  Production wrappers must redact client IDs, tokens, prompts, model paths, endpoint IPs/hosts, CA paths, and certificate pins from user-visible or remote logs.
